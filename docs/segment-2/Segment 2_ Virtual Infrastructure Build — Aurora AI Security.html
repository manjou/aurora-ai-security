1 Executive Summary & Scope

This document describes the virtual infrastructure build for the Aurora AI Security GmbH lab environment, completed as part of Segment 2 of the Masterschool Cybersecurity Bootcamp.

The lab simulates a miniaturized subset of the Aurora production network, demonstrating VLAN segmentation, inter-VLAN routing, firewall enforcement, and centralized SIEM monitoring — all on a single MacBook Air M3 using the UTM hypervisor.

Simulation vs. Production Deviations

IP Addressing: Production uses 10.10.x.x (Freiburg HQ). The simulation uses 172.16.x.x to avoid conflicts with UTM's NAT range (192.168.64.0/24) and to clearly separate lab traffic from hypervisor management traffic. The third octet mirrors the VLAN ID in both schemes (e.g., VLAN 30 → x.x.30.x).
SIEM Platform: Production specifies Splunk / Microsoft Sentinel. The simulation uses Wazuh 4.14.3 as the open-source equivalent, providing identical functionality (log collection, indexing, alerting, dashboards) within lab resource constraints.
VLAN Technology: UTM does not support 802.1Q VLAN tagging. The solution uses separate virtual network bridges per VLAN — one UTM bridge per segment. pfSense receives one NIC per bridge, enforcing real inter-VLAN routing at Layer 3.

9 Extension: Syslog Centralization
Extension Credit: This section implements the "Syslog Centralization" extension from the Segment 2 advanced tasks — deploying pfSense log forwarding to Wazuh as a syslog server.

Architecture

pfSense forwards system and firewall logs to Wazuh Manager via UDP syslog on port 514. Wazuh listens for syslog on its VLAN 30 interface and processes incoming pfSense events through its detection engine.

ComponentConfiguration
pfSense syslog destination172.16.30.10, UDP 514
Log categories forwardedSystem Events, Firewall Events
Wazuh listenerossec.conf — second <remote> block, connection=syslog, port=514, protocol=udp
Allowed source IP172.16.30.1 (pfSense VLAN30 interface)
Wazuh port bindingwazuh-remoted listening on 0.0.0.0:514 UDP (verified via ss -ulnp)

Wazuh ossec.conf — Remote Block

<!-- Agent communication (existing) -->
<remote>
  <connection>secure</connection>
  <port>1514</port>
  <protocol>tcp</protocol>
  <queue_size>131072</queue_size>
</remote>

<!-- pfSense syslog listener (added for extension) -->
<remote>
  <connection>syslog</connection>
  <port>514</port>
  <protocol>udp</protocol>
  <allowed-ips>172.16.30.1</allowed-ips>
</remote>
